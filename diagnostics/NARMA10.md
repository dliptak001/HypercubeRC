# NARMA-10 Nonlinear Benchmark

## What this diagnostic measures

NARMA-10 (Nonlinear AutoRegressive Moving Average, order 10) is the
standard reservoir computing benchmark for combined memory and nonlinear
computation. The target sequence is generated by the recurrence:

    y(t+1) = 0.3*y(t) + 0.05*y(t)*sum(y(t-i), i=0..9) + 1.5*u(t-9)*u(t) + 0.1

where u(t) is a random input in [0, 0.5].

**The task:** given the input u(t), predict the target y(t) from reservoir
states. The readout must reconstruct a signal that depends on 10 steps of
history and nonlinear products of past values.

**The metric:** NRMSE. Standard ESN results range from 0.2 to 0.4.

## Why this benchmark matters

NARMA-10 is harder than Mackey-Glass because it demands three capabilities
simultaneously:

1. **Memory depth.** The u(t-9)*u(t) term requires remembering input from
   10 steps ago. A reservoir with short fading memory will fail here.

2. **Nonlinear computation.** The y(t)*sum(...) and u(t-9)*u(t) terms are
   products — a purely linear system cannot compute them. The reservoir's
   tanh nonlinearity must encode these interactions in its state vector.

3. **Temporal integration.** The autoregressive y(t) feedback means the
   target depends on its own history, not just the input history.

A reservoir that can only remember (high MC) OR only compute nonlinearly
will fail — it must do both. This is where the translation layer has its
biggest impact: the x² and x*x' features give the linear readout direct
access to the quadratic interactions that the NARMA-10 recurrence demands.

## How it works

1. Generate NARMA-10 input/target sequences (deterministic per seed).
2. Scale inputs to [-1, +1] for reservoir injection (raw NARMA inputs
   are in [0, 0.5]).
3. For each of 3 seeds {42, 1042, 2042}:
   - Run with raw features (N selected states).
   - Run with translation features (2.5N).
   - Train the selected readout on 70%, test on 30%.
4. Report 3-seed average NRMSE for raw and translation, plus % change.

## Sample results

Run with Ridge Readout, 3-seed average:

| DIM | N    | Raw   | Full Translation | Change |
|-----|------|-------|------------------|--------|
| 5   | 32   | 0.566 | 0.539            | -4.7%  |
| 6   | 64   | 0.417 | 0.264            | -36.7% |
| 7   | 128  | 0.387 | 0.176            | -54.6% |
| 8   | 256  | 0.399 | 0.125            | -68.6% |

## What to look for

- **Translation dominates at DIM >= 6.** Improvement scales dramatically:
  -37% at DIM=6, -55% at DIM=7, -69% at DIM=8. The quadratic features
  are essential for reconstructing the product terms in the NARMA recurrence.

- **Raw features plateau.** Without translation, NRMSE stalls around 0.4
  at DIM 7-8 because the linear readout can't access the nonlinear state
  interactions. Adding more neurons doesn't help — the bottleneck is the
  readout, not the reservoir.

- **Translation breaks through the plateau.** With 2.5N features, NRMSE
  keeps improving with DIM. At DIM=8, translation reaches 0.125 — well
  below the standard ESN range of 0.2-0.4.

- **This is where the translation layer earns its keep.** On Mackey-Glass
  (a smoother task), translation helps 20-35%. On NARMA-10, it helps
  35-70%. The harder the nonlinear computation, the more the quadratic
  features matter.
