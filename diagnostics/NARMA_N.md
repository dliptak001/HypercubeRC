# NARMA-N Nonlinear Benchmark

## What this diagnostic measures

NARMA-N (Nonlinear AutoRegressive Moving Average, order N) is the standard
reservoir computing benchmark for combined memory and nonlinear computation.
The target sequence is generated by the recurrence:

    y(t) = alpha*y(t-1) + beta*y(t-1)*sum(y(t-1..t-N))
           + gamma*u(t-N)*u(t) + delta

with the canonical coefficients `alpha=0.3, beta=0.05, gamma=1.5` and
inputs `u(t)` drawn uniform in `[0, 0.5]`. Order **N = 10** is the canonical
RC reference task; higher N stresses longer memory horizons.

**The task:** given the input u(t), predict the target y(t) from reservoir
states. The readout must reconstruct a signal that depends on N steps of
history and nonlinear products of past values.

**The metric:** NRMSE. Standard ESN results range from 0.2 to 0.4.

## Why this benchmark matters

NARMA-N demands three capabilities simultaneously:

1. **Memory depth.** The `u(t-N)*u(t)` term requires remembering input from
   N steps ago. A reservoir with short fading memory will fail here.

2. **Nonlinear computation.** The `y(t-1)*sum(...)` and `u(t-N)*u(t)` terms
   are products — a purely linear system cannot compute them. The reservoir's
   tanh nonlinearity must encode these interactions in its state vector.

3. **Temporal integration.** The autoregressive `y(t-1)` feedback means the
   target depends on its own history, not just the input history.

A reservoir that can only remember (high MC) OR only compute nonlinearly
will fail — it must do both.

## Stability: the `use_tanh` flag

The `beta*y(t-1)*sum(...)` term is positive feedback: as y grows the sum
grows, which makes y grow faster. Left unbounded the series can diverge,
and the risk increases with order N.

`NARMA_N_Generator` exposes a `use_tanh` flag rather than baking in a bound:

- **`use_tanh = false`** — the raw recurrence runs unbounded. Canonical
  NARMA-10 with inputs in `[0, 0.5]` stays bounded on its own, so this is
  the literature-faithful choice for N = 10.
- **`use_tanh = true`** — each y is squashed through `tanh`: a smooth,
  saturating bound with no information-destroying kink. This is the
  established convention for higher-order NARMA in the RC literature and
  should be enabled for larger N. The benchmark also shrinks `delta` to
  0.01 for N >= 20 to keep the series in a usable range.

This replaces the earlier hard clamp to `[0, 1]`, which produced flat
saturated stretches that deflated NRMSE and matched no standard variant.

## How it works

1. Generate a NARMA-N input/target series (deterministic per seed) via
   `NARMA_N_Generator`. The generator is O(1) per step — an incremental
   running sum and deque ring buffers replace per-step window recomputation.
2. Scale inputs to [-1, +1] for reservoir injection (raw NARMA inputs
   are in [0, 0.5]).
3. Run the reservoir and collect raw states (N = 2^DIM floats per step).
4. Train the HCNN readout on 70%, test on 30%.
5. Report NRMSE.

Uses a single per-DIM seed selected by 500-seed survey (`SurveyedSeed<DIM>()`).
`NARMA_N<DIM>` defaults to order 10 with `use_tanh = false`; pass a larger
`narma_order` and `use_tanh = true` to run longer-horizon variants.

## Sample results

Run with HCNN readout (`Baseline<DIM>()` config), canonical NARMA-10:

| DIM | N    | HCNN NRMSE |
|-----|------|------------|
| 7   | 128  | 0.188      |
| 8   | 256  | 0.118      |
| 9   | 512  | 0.096      |
| 10  | 1024 | 0.084      |

> These supersede the prior table (0.218 / 0.153 / 0.134 / 0.122), which
> was produced by the old `GenerateNARMA10` generator. The current
> generator uses a different RNG stream and — more importantly — fixed a
> target-alignment bug (see the `NARMA_N_Generator` header comment): the
> source version paired `u(t)` with `y(t+1)`, an unlearnable one-step
> forecast that had pinned NRMSE near 0.8.

## What to look for

- **NRMSE improves with DIM.** More neurons = deeper memory + richer
  dynamics. The HCNN readout discovers nonlinear features via learned
  convolution, breaking through the plateau that linear readouts hit.

- **HCNN learns the quadratic interactions** that the NARMA-N recurrence
  demands (product terms `u(t-N)*u(t)` and `y(t-1)*Σy`), without
  hand-crafted feature engineering.
